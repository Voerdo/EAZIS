На главную

Быстрый старт

1. На главной странице перетащите DOC или DOCX файл в область загрузки или нажмите на неё для выбора файла.

2. Дождитесь окончания анализа — система автоматически извлечёт текст, разобьёт его на предложения и построит синтаксические деревья.

3. Нажмите «Открыть» рядом с загруженным документом, чтобы увидеть список предложений.

4. Для каждого предложения доступны две визуализации:

  • Дерево зависимостей — показывает бинарные отношения между словами (главное → зависимое)
  • Дерево составляющих — показывает иерархическую группировку слов во фразы (NP, VP, PP и т.д.)

Дерево зависимостей

Дерево зависимостей показывает, как слова в предложении связаны друг с другом бинарными отношениями «главное → зависимое».

Режимы работы:

  • Выбор — кликните на слово, чтобы увидеть и отредактировать его свойства (лемма, часть речи, член предложения)
  • Перемещение — перетаскивайте слова для изменения их расположения на холсте
  • Связь — создавайте новые синтаксические связи между словами, кликая сначала на главное слово, затем на зависимое

Управление:

  • Колёсико мыши — масштабирование
  • Ctrl + зажатая ЛКМ — панорамирование холста
  • Двойной клик по связи — удаление связи
  • Кнопки + / − / ⟲ — масштабирование и сброс вида

Редактирование:

Вы можете редактировать свойства слов и типы связей. Все изменения сохраняются в базу данных при нажатии кнопки Сохранить в БД.

Дерево составляющих

Дерево составляющих (constituency tree) показывает иерархическую группировку слов во фразы (фразовые категории). В отличие от дерева зависимостей, оно не показывает стрелки между отдельными словами, а объединяет их в блоки.

Основные фразовые категории:
NP (Noun Phrase) — именная группа
VP (Verb Phrase) — глагольная группа
PP (Prepositional Phrase) — предложная группа
ADJP (Adjective Phrase) — группа прилагательного
ADVP (Adverb Phrase) — группа наречия
S — предложение

Дерево отображается в горизонтальном формате с ветвями для наглядности. Вы можете экспортировать его в текстовый файл.

️ База данных

Все данные хранятся в SQLite базе syntax_analyzer.db:

  • documents — загруженные документы (имя, дата, текст, размер)
  • sentences — предложения из документов
  • tokens — токены (слова) с морфологической и синтаксической информацией
  • constituency_trees — деревья составляющих в скобочной нотации
